CSS Selectors

  • * : universal selector
  • element selector: selects all elements of a given type. Uses the html elements
  • You can use a comma as an "or" between to selectors
  • ID selectors: we can associate an ID (as an argument) to any element. To provide a hook for css to single out a single element. You should have only one object for any ID You wanna use as little ID as possible. Classes are better
  • class selector. A class is like an ID for multiple elements. For example you can tag alla spans with a certain class. It makes sense to give the parent elements classes and use the descendant selector to select things inside of them.
  • descendant selectors: a space. A space means that the second element i nested inside of the first
  • adjacent combinator: the + is used to select elements on the same level, one after the other. Like h2 + button
  • Direct descendant combinator: > is used to select an element that is directly nested inside of another. For example li > a selects all anchor tags inside of a point-of-list element
  • Attribute selectors: allows us to select elements with a certain attribute set to a specific value. It works like this: element[attribute="value"] And you can use also fuzzier matches: with *= instead of the equal sign you get all the elements that have a value for that element that contains the value we give. Look at the MDN page.
  • pseudo classes: they identify a state of an element. They all start with a column. For example a:hover selects anchor tags that are being hovered over.
    • :hover
    • :active, for something that is currently being activated (a button that is clicked and held.
    • :checked
    • :n-th-of-type(mn), m is a number. n has to stay as letter.
  • pseudo-elements. You define them wit ::
    They select a particular part of an element
    • ::first-letter
    • ::first-line
    • ::selection applies to highlighted parts of elements
