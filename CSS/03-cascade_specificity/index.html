Cascade and Specificity

Order matters

In our css files, order matters. Also, if I link more style-sheets, the order I include them matters.

Specificity

What happens when you have more selectors selecting the same object? Which one wins? When I have a conflict the most specific wins. And there is an actual formula for that:
ID > CLASS > ELEMENT IDs count like infinite to the second, classes like inf to the first, elements like infinite to the zero. Anyway you can use a specificity calculator. pseudoclasses count as classes, pseudoelements count as elements. This applies only when we have a conflict: if different properties are set in different blocks, no conflict arises and they are all applied.

Inheritance

Some css properties will be inherited by child elements, if not set specifically. Certain elements don't inherit by default.
