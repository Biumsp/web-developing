What is HTML5?

It is not a version. It is the latest evolution of the standard that defines HTML. It is a version of the language and the newest set of technology related to HTML. Like stuff for css, or 3d graphics... a lot of thing. There is a standard because there is not one official version, there exists only the rules of HTML, and every browser implements some HTML interpreter based on that standard.

The declaration we put on the top is to tell the browser that we are using HTML5 semantics, elements, standards...

Inline vs. Block elements

This terms refer to how a given element fits in with the flow of content in the document.

Paragraphs are block elements: they take up a whole block (you get a new line after them).

Anchor tags, images.. are inline elements: they don't take up a whole line.

We can change this behavior using CSS.

Div

  • It is a generic container to group things together.
  • It is a block element
  • It is a way to style things together using CSS

Spans

Spans are also grouping elements, like divs, but they are inline elements. So, they will take up as little space as possible. They are very useful, for example, to style text in between a paragraph. If I want to change the color of some words, I can use a span to wrap them.

HTML Entities

There are characters that are reserved for HTML, or they are too complex to type.

They all start with &# and end with ;
with a number in between. We also have another syntax, where we have some text in place of the number: in this case there is no hashtag. You can find a list here

For example:

  • The less than will have "lt" as text: <
  • The greater than will have "gt" as text: >
  • The ampersand will have "amp" as text: &

Semantic Markup

It is the concept of writing the html structure in a meaningful way. For example, using divs for every grouping, is terrible. There are more meaningful elements with the exact same functionalities. For example:

  • nav: for navigation bar
  • footer and header
  • main element
  • section: exactly like a div
It matters to help websites crawlers and for accessibility. There are also elements equivalent to nav or paragraph (p), but with a different and more meaningful name.

"Main" element

The content of the main element should be unique to a document. Content that is repeated across multiple pages should not be included (like nav bars, logos, forms...).

Nav element

Used to group sections with navigation links

Section element

This rapresents a standalone section of a document. It is used for sections that have not a more specific meaning than "section".

Aside element

Something that is not strictly needed in the main part of the document. It is kind of a side note, or a side element..

Time element

An inline element that is used to call something out as "time" in a machine readable format
