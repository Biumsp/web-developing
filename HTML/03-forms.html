Forms

The "action" attribute tells where to send the data when the form is submitted. We can use an HTTP request to sand the data. The "method" attribute is used to define which kind of HTTP request to use.

Input element

The input element can be used to get 20+ input types, depending on the argument "type". It does not have a closing tag. We have all the options on the MDN page. We can define a placeholder.

Labels

Labels are elements that we associate to the input elements. They have two functions:

  1. They provide some descriptive text to the input element
  2. They are linked to the input element. A div wouldn't be, so it's better for accessibility
  3. It allows to click on the label to select the input
We connect it to an input by giving the input an ID. Then we give the label an argument "for" with value corresponding to the ID of the input.

Buttons

We can have buttons inside or outside of a form. When I have a button in a form it will submit the form when clicked, by default. If I have them outside of the form it does nothing by default. We can set the type of the button: if we set it to "button" instead of "submit" it does not submit.

Name attribute of forms

The name attribute is the way we label inputs in our HTTP request.

Other input types

We have many other input types. The most common are:

  • checkbox: works like the others, but you can select if it is checked by default
  • radio: is for radiobuttons. They are still y/n buttons, but in a group you can only have une checked. You group them together by giving them the same name. We have to add the argument "value" to tell te brawser what to send to the server, otherwise only "on" is sent.
  • range: creates a slider to select a value in a range

The select element

This is not a type of input: it is another different element. You use it to make a drop-menu. It is the combination of two elements: the select element, which is the parent one, and the option elements.

Textarea

This is a multiline input element, whee you can type text

Forms validation

Validating user inputs: for example requiring that something is checked, or a valid password. We can do it with JS, but there is also the built in validation in the browser. We also have server-side validation: when we submit some data and we check, for example, if the username is valid. In built-in validation we can:

  • Set some inputs to "required", by adding this attribute. I can't submit if it is not set.
  • Set min and max len of text. For example a password
We have the possibility to add regex patterns to match. Some elements have a default validation (it could be pattern matching). For example I can set an input type to "email" and it will check if it matches an email pattern. Or type="url" checks if it is a url. And you can set the url pattern.
